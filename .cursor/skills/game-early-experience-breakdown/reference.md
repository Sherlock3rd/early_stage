# 游戏前期体验拆解参考

## 1. 单一事实源

每个游戏的 `analysis.json` 是该拆解唯一分析事实源。`build_viewer.py` 只将图片路径重写到
`data/<dataset-id>.json` 与 `screenshots/<dataset-id>/`；`write_feishu_sheet.py`
从同一份已校验数据生成页签。禁止分别维护 HTML 文案和飞书文案，也禁止多个游戏共用一个 JSON。

文件使用 UTF-8 JSON。以 `analysis_model.py` 的校验器为最终机器契约；本文补充语义约束。

## 2. `analysis.json` 契约

根节点：

```json
{
  "video": {
    "path": "recordings/game.mp4",
    "duration_seconds": 8237.0
  },
  "slices": [],
  "global_curves": {
    "scale": {"min": 0, "max": 5},
    "points": []
  },
  "global_loops": {
    "scope": {},
    "macro_loops": [],
    "loop_families": [],
    "nodes": [],
    "edges": []
  }
}
```

- `"video.path"`：非空录屏路径。
- `"video.duration_seconds"`：大于 0 的有限秒数。
- `"slices"`：非空数组；索引从 0 连续递增，时间片从 0 连续覆盖完整视频。
- `"global_curves"`：必填的人工情绪强度、体验强度与体验趋势数据。
- `"global_loops"`：必填的全局玩家闭环和线性入口/出口流程。

每个时间片必须包含且保留以下字段：

```json
{
  "index": 0,
  "start": 0.0,
  "end": 60.0,
  "main_frame": {
    "path": "frames/slice-000-main-000030000.jpg",
    "timestamp": 30.0
  },
  "evidence_frames": [
    {
      "path": "frames/slice-000-evidence-01-000012000.jpg",
      "timestamp": 12.0
    }
  ],
  "dimensions": {
    "阶段目标": {"fact": "熟悉移动并抵达营地。", "inference": "本片建立基础操作认知。"},
    "任务链": {"fact": "Tutorial: 前往营地", "inference": "主要任务是完成移动教学。"},
    "核心循环": {"fact": "[New] T0 移动 (寻路)", "inference": "系统教学移动与目标点反馈。"},
    "渐进体验": {"fact": "移动/寻路", "inference": "首次解锁移动操作。"},
    "地图体验": {"fact": "X玩法低镜头锁定区域", "inference": "首次开放开场锁定区域。"},
    "经济体验": {"fact": "", "inference": ""},
    "剧情轴": {"fact": "导师提出营地遭遇危机。\n目的：建立玩家前往营地的动机。", "inference": "冲突铺垫。"}
  },
  "stage_range": {
    "stage_id": "tutorial-movement",
    "name": "基础移动教学",
    "start": 0.0,
    "end": 180.0
  },
  "narrative_climax": {
    "judgement": "none",
    "reason": "00:12 仅建立冲突；证据 frames/slice-000-evidence-01-000012000.jpg"
  },
  "flow": {
    "judgement": "none",
    "reason": "操作单一，尚无明确挑战—能力匹配高点"
  },
  "confidence": 0.82,
  "evidence": [
    {
      "frame": "frames/slice-000-evidence-01-000012000.jpg",
      "note": "00:12 字幕与导师对话首次提出危机"
    }
  ],
  "open_questions": [
    "导师语音中的专有名词需用户确认"
  ]
}
```

字段规则：

- `"start"`/`"end"` 为左闭右开秒数区间；边界必须与 `video_timeline.py` 完全一致。
- `"main_frame.timestamp"` 默认严格等于时间片中点。若中点无信息，仅当画面为纯黑、纯白或
  白屏/迷雾过场等无法传达体验内容时，才可改用片内距离中点最近的临近有效帧，并增加
  `"selection_reason": "midpoint_uninformative"`。允许偏移上限为 `min(30 秒, max(10 秒, 片长 × 10%))`；
  不能仅因构图或审美偏好偏移，也不能只填写理由字符串来绕过临近范围校验。
- `"evidence_frames"`：非关键片可为空；关键转折必须有 1-3 张，时间戳位于所属片内。
- 图片路径必须是相对 `analysis.json` 的安全相对路径，文件存在、非空，且魔数确认为 PNG、JPEG、GIF 或 WebP。
- `"dimensions"` 必须恰好含七个键；每项必须是含字符串 `"fact"` 和 `"inference"` 的对象。
  没有有效信息时允许使用空字符串，禁止用“未观察到”填充飞书。
- `"stage_range"` 必须含稳定 `"stage_id"`；相邻同 ID 的名称与范围完全一致，不同阶段按时间片边界连续衔接。
- `"narrative_climax.judgement"` 只允许 `none`、`climax`、`low`。
- `"flow.judgement"` 只允许 `none`、`flow_peak`。
- `"confidence"`：0-1；表示本片综合判断置信度，模糊画面/数值/音频必须降低。
- `"evidence"`：`frame` 必须引用该时间片的主图或 1–3 张证据图之一；`note` 包含时间与可观察事实。
- `"open_questions"`：只放待确认项；没有时用空数组。

### 全局情绪与体验曲线契约

`global_curves.points` 与现有时间片一一对应，数量、顺序、`start`、`end` 必须与 `slices`
完全一致。0–30 分钟沿用每分钟评分点，30–60 分钟沿用每 5 分钟评分点，1 小时后沿用每
10 分钟评分点，不插值制造更细展示数据。展示点必须聚合密集分析层的真实发现；片内存在
短时关键剧情时，情绪子分取该片最强剧情刺激，不能用中点画面状态覆盖高潮。

每个点包含：

```json
{
  "start": 0,
  "end": 60,
  "emotion": {
    "narrative_score": 1,
    "supporting_score": 3,
    "intensity": 1.6,
    "valence": "negative",
    "drivers": ["narrative", "environment_pressure", "urgency"],
    "event": "暴风雪倒计时持续逼近",
    "reason": "轻度剧情铺垫叠加持续环境威胁"
  },
  "experience": {
    "score": 2,
    "basis": {
      "gameplay_concentration": "以剧情过场为主，主动操作较少",
      "feedback_density": "获得新任务反馈",
      "goal_challenge": "后续目标变得明确",
      "interruption": "过场中断玩法操作"
    },
    "summary": "剧情推动强，但玩法沉浸强度较低"
  }
}
```

- `emotion.narrative_score` 为 `0～5` 的剧情刺激子分；`emotion.supporting_score` 为 `0～5`
  的其他刺激子分。其他刺激包括环境压力、暴风雪倒计时、紧迫目标、战斗冲突、成长奖励和危机缓解。
- 情绪曲线以剧情为主体，`emotion.intensity` 固定为
  `round(0.7 × narrative_score + 0.3 × max(narrative_score, supporting_score), 1)`。
  因此剧情 5 分仍为最终 5 分，纯环境/系统刺激最高贡献 1.5 分。
- `emotion.drivers` 是无重复多选数组，只允许 `narrative`、`environment_pressure`、`urgency`、
  `combat`、`progression_reward`、`relief`。剧情子分大于 0 时必须包含 `narrative`；
  剧情子分为 0 时不得包含 `narrative`；其他刺激子分大于 0 时必须包含至少一个非剧情来源，
  为 0 时不得包含非剧情来源。
- `emotion.valence` 只允许 `positive`、`negative`、`mixed`、`neutral`。负向高峰不代表剧情质量差。
- 情绪强度大于 0 时，`event` 与 `reason` 必须为非空文本；`event` 描述可观察的主要情绪刺激，
  不再只描述剧情事件。强度为 0 时允许空置。
- `experience.score` 为 `0～5` 的人工玩法沉浸/心流强度。
- `experience.basis` 必须包含非空的 `gameplay_concentration`、`feedback_density`、
  `goal_challenge`、`interruption`，并用 `summary` 总结评分原因。
- 七维事实继续保留，但不参与双曲线评分或加权。

体验实际折线继续使用全部时间片的 `experience.score`。体验整体趋势先按
`stage_range.stage_id` 聚合阶段平均分，每个阶段在其真实起止时间中点形成一个等权观测值，
再做普通最小二乘线性回归；这样保留片内摩擦与低谷，同时避免长阶段仅因时间片更多而对宏观趋势
形成过度权重。无大地图里程碑时，回归线从首个评分点绘制到最后一个评分点；首尾预测值之差 `>=0.5` 为上升、
`<=-0.5` 为下降，其余为持平。若存在 `type=slg_entry` 的 `timeline_milestones`，回归训练范围与
绘制终点固定为首次大地图入口，并包含入口所属时间片；入口之后保留情绪和体验实际折线，但不参与
宏观趋势。预测显示截断在 `0～5`，趋势不得修改原始人工分数。

### 全局 LOOP 流程契约

`global_loops` 用于回答玩家为何投入、执行什么、获得什么，以及多个小闭环如何按时间串成大循环。
只有完整形成“明确动机 → 连续行为 → 即时奖励 → 下一动机/再次投入”的内容才允许使用
`micro_loop`；系统教学步骤属于小 LOOP 的行为，不单独成环。一次性剧情和范围出口使用线性节点。

- `scope` 必含 `start`、`end`、`end_label`、`outside_exit_label`，主体范围必须位于视频时长内。
- `macro_loops` 为非空数组；每项包含唯一 `id`、`title`、`summary` 和唯一
  `settlement/expedition/hero_growth` 配色。大 LOOP 仅负责分组，不替代小 LOOP。
- `loop_families` 为非空数组；每项包含唯一 `id`、`title`、`summary` 和唯一 `accent`。
  类型由分析者按玩法语义显式定义并且必须被至少一个小 LOOP 使用，不得根据标题相似度自动归类。
- `nodes` 的 `type` 只允许 `micro_loop`、`transition`、`end`、`outside_exit`，并按最早关联时间片排序。
- 每个节点必含唯一 `id`、标题、摘要、`macro_loop_id`、非空 `slice_indices`、非空
  `evidence_frames` 和 `confirmed/pending_confirmation` 状态。
- 主体节点的关联时间片必须完整位于 `scope`；`outside_exit` 最多一个，且必须只引用主体结束后的首个时间片。
- 节点证据只能引用关联时间片已有主图或证据图。
- `micro_loop` 的 `macro_loop_id` 必须引用一个大 LOOP，`loop_family_id` 必须引用一个小 LOOP 类型，
  并额外包含 `0～1` 的 `confidence`（必须取关联时间片最低值）与非空 `motivation`、`behaviors`、
  `reward`、`next_motivation`；其他节点的 `macro_loop_id` 必须为空且不得携带闭环或类型字段。
- 每个小 LOOP 只能有一个主要大 LOOP；跨系统影响不得通过多个归属表达。
- `edges` 必含 `from`、`to`、`kind`、`label`；`kind` 只允许 `primary`、`macro_return`、
  `cross_macro`、`conditional`。
- `primary` 必须按节点顺序连续覆盖全部主体节点且不得时间逆序。
- `macro_return` 两端必须是同一大 LOOP 的小 LOOP，且中间确有其他大 LOOP 或线性节点打断。
- `cross_macro` 两端必须是不同大 LOOP 的小 LOOP；HTML 从来源奖励/下一动机连接到目标动机。
- `outside_exit` 只能由主体终点通过 `conditional` 连线进入，且不得再连接回主体流程。
- 旧 `chapters`、`entry`、`loop`、节点内 `accent` 和 `feedback` 口径不再使用；旧分析文件必须迁移。

HTML 在时间片详情之后、页面最后按实际时间渲染一条主链。每个小 LOOP 直接展开四节点；
聚落建设、战斗远征、英雄养成分别用青蓝、橙红、紫色分段背景带包住连续小 LOOP。
桌面绘制同色回流线和跨环 SVG 箭头，移动端用完整“起点 → 终点：关系”文本替代大范围连线。
图例只淡化大 LOOP，不删除或重排节点。四段内容直接常驻卡片，LOOP 区不生成 Tooltip；
`evidence_frames` 保留在数据中用于校验和追溯，但不在该区域展示。点击卡片联动首个可见时间片。

标题下、图例上方展示由节点实时派生的统计：事件级小 LOOP 总数、`loop_family_id` 去重数，
以及每类 `出现次数` 和 `强化次数 = max(出现次数 - 1, 0)`。统计不可手填，也不受大 LOOP 淡化影响。
时间片详情展示所有直接关联的 `micro_loop` 标签；点击标签滚动至对应卡片并短暂高亮。
线性节点、终点和图外出口不得进入详情标签；无关联时显示“当前时间片无关联 LOOP”。

寒霜主体固定为 17 个事件级小 LOOP：聚落 12、远征 3、英雄 2；按玩法语义去重为 6 类：
建筑升级养成 8 次、建筑生产 2 次、推关玩法 3 次、英雄养成 2 次、法令系统 1 次、
临时供暖强化 1 次。P8 完整主城解锁（25:00）为主体终点，
P9 进入 SLG 大地图只作为图外虚线出口，具体解锁门槛未有证据时保持 `pending_confirmation`。

### 事实、推断、置信度、待确认

- `dimensions` 用对象字段分离 `fact` 与 `inference`。`fact` 是飞书与 HTML 最终展示文案；
  `inference` 只服务内部分析，不进入交付页面。没有可靠事实时两项允许空置，不要制造推断。
- `evidence[].note` 只描述可观察内容，不写结论。
- `confidence` 只表示把握程度，不能替代事实来源。
- 不确定的名称、数值、因果、版本差异写入 `open_questions`。

### 剧情轴展示与聚合索引

- `dimensions["剧情轴"]` 是当前时间片展示文本，描述该片内可见的剧情事实与谨慎推断。
- 独立的 `"narrative_climax"` 与 `"flow"` 是跨片聚合索引，用于从完整时间线上定位高潮、
  低谷和心流高点，不是第二套剧情文案。
- 时间片展示与跨片聚合索引必须引用 `evidence` 中的同一证据、时间和观察事实，结论不得矛盾。
- 聚合判断需要比较相邻及前后时间片；不能仅凭单张截图把普通事件标为高潮或心流高点。

## 3. 七维口径

以下格式来自参考表 `ai学习` 页签，目标是让策划横向扫读各时间片，而不是展示分析过程。

1. **阶段目标 (Experience Goal)**：格式为 `P序号. 阶段名 (English)`，下一行用一句话写阶段体验目标。
   连续时间片合并，用于传达宏观节奏与阶段终点。
2. **任务链 (Quest Chain)**：只写该时间片玩家的主要任务。推荐
   `Intro/Tutorial/Main/Unlock: 任务名 (可选时间)`；不是点击、领奖、返回等逐步操作日志。
3. **核心循环 (Core Loop)**：描述系统对玩家的教学及循环环节。格式为
   `[New]/[Upgrade]/[Loop]/[Cont]/[End] + T编号 + 系统 + (教学环节)`。
4. **渐进体验预期 (New Content)**：只回答“期间解锁了什么系统、机制或界面”，使用短名词组；
   没有新内容时允许空置。
5. **地图体验预期 (Map Progress)**：只回答“期间解锁了哪些地图或区域”，不写镜头、氛围、
   路线感受等评价。寒霜按 `X玩法低镜头锁定区域`、`X玩法全局区域`、`主城全区域`、
   `SLG大地图` 四级记录；没有地图变化时允许空置。
6. **经济体验预期 (Eco Progress)**：只记录玩家等级/战力值/主城等级，以及用户指定的核心建筑等级
   等成长快照，字段换行展示。寒霜只补充大熔炉等级，不扩展到全部功能建筑。
   只记录可辨识的实际等级：界面显示 `大熔炉 Lv.6` 才能记录 Lv.6；
   “需要大熔炉等级 4”或“升级到 6（5/6）”是门槛/进度，升级条件不等于当前等级。
   不写资源来源、消耗、价格或经济评价；看不清数值时允许空置或待确认。
7. **剧情轴**：只写“这段剧情在讲什么”以及“叙事目的”。推荐
   `剧情事件\n目的：为什么在此处讲这段剧情`；没有剧情推进时允许空置。

### 地图层级的证据规则

- 不能仅凭镜头远近、建筑数量或画面变大判断地图层级。
- `X玩法` 模拟体验区可包含生存建筑与升级操作，但缺少 SLG 完整功能建筑集合，例如造兵、
  科研、联盟等；这类画面仍不能标为主城全区域。
- `主城全区域` 必须以完整主城功能和城市总览实际开放为证据。
- `SLG大地图` 必须出现世界坐标、行军、其他城镇或世界地图交互等证据。
- 地图切换时查看变化前、解锁瞬间和变化后三段画面，避免把模拟体验区的视角变化误写成新地图。

### 关键剧情的证据规则

- 对话、旁白、墓碑、书信、回忆和转场构成同一关键过场时，必须查看完整片段与连续字幕。
- 死亡、离别、背叛、牺牲、重大胜利、身份揭示和继承目标不得因分钟中点图没有命中而遗漏。
- 背景设定卡、人物档案、世界观说明和旁白式文本默认属于“设定信息”，不是片内已发生事件。
  只有连续画面、对话、行动与结果能闭合事件因果时，才能写为实际剧情。
- 女性立绘、黑底文字、坐姿、倒地或疑似跪倒等单一视觉信号不能独立证明死亡、离别、牺牲或
  重大转折，也不能独立触发 `narrative_climax=climax`。
- 剧情轴可以用 2–4 句交代人物、事件、因果与叙事目的；避免只写“建立动机”等抽象结论。
- 关键剧情高点同时写入 `narrative_climax`，并引用最能证明事件的片内证据图。

剧情高潮与心流高点不是同义词。高潮是叙事张力峰值；心流高点要求挑战与能力匹配、
目标清晰、反馈及时。等待、重复、迷失或反馈中断可构成低谷。

`narrative_climax` 与 `flow` 继续作为 HTML 内部聚合索引；飞书剧情轴不显示“高潮/低谷：无”
或“心流：无”，也不展示“事实：”“推断：”前缀。

## 4. 时间与截图规则

- 分析层审阅点：`0 <= t < 10m` 每 1 秒，`10m <= t < 20m` 每 5 秒，
  `t >= 20m` 每 10 秒。该层负责发现事件，不直接成为 HTML 节点。
- `0 <= start < 30m`：每 1m 一片。
- `30m <= start < 60m`：每 5m 一片。
- `start >= 1h`：每 10m 一片。
- 最后一片截到视频实际结束，不向上补齐。
- 主截图默认取 `(start + end) / 2`，保证可重复。中点无信息时按上述规则改选临近有效帧并记录
  `midpoint_uninformative`，同时复核替换后画面确实包含角色、界面、剧情或地图等可读内容。
- 关键转折先由主图与原视频确认，再选择最接近“变化前/触发时/结果后”的 1-3 个绝对秒数。
- 密集审阅点出现对话、CG、倒地、死亡、离别、牺牲、身份揭示、重大胜利或章节转场时，
  必须连续查看候选点前后原视频，并以 0.5～1 秒精度记录事件；禁止只查看相邻两个采样点。
- 同一片只保留支持判断的最小证据集；中点图不能证明转折时必须补图。
- OCR 只辅助读取文字。动作顺序、交互反馈、音效、语气、过场和上下文必须通过画面及原视频确认。
- 验收要求是查看全部主截图及必要原视频片段；无需线性观看无信息片段。

## 5. HTML 布局

固定包：

```text
viewer/
  index.html
  data/
    datasets.json
    frost.json
    sanbing.json
  screenshots/
    frost/
    sanbing/
```

- `--dataset-id` 使用稳定的小写英文 slug；每个拆解独占一个 JSON 和同名截图子目录。
- `--dataset-name` 写入 `data/datasets.json` 的显示名；构建同名 ID 时更新标签，构建新 ID
  时保留已有条目、JSON 和截图。
- 对同一输出目录构建新 `dataset-id` 时保留已有数据集；重建同名 ID 时原子替换该 ID 的
  JSON 与截图，避免遗留旧帧。
- 页面通过 `?dataset=<dataset-id>` 选择数据；未提供参数时读取构建时写入
  `analysis-data-file` 元信息的默认数据。
- 数据集参数只允许小写字母、数字与连字符，禁止路径字符和目录穿越。
- 标题区右上角提供“拆解项目”下拉框并读取 `data/datasets.json`；切换时更新安全的
  `dataset` 参数并重载目标数据，避免残留上一项目状态。
- 顶部提供标题、视频时长和数据集切换；不提供阶段、关键节点或关键词筛选。
- 主区使用单一总览时间轴代替七条泳道，所有时间片按真实时间比例压缩在容器宽度内；
  支持点击节点、上一片/下一片、阶段快速跳转和键盘左右方向键，不产生页面横向滚动。
- 根节点可包含 `timeline_milestones`，每项固定为
  `{id, type, label, timestamp, slice_index, note}`；`type` 支持 `slg_entry`、`map_entry` 和
  `cg_end`。`slg_entry` 仅用于出现世界坐标、据点/城镇、行军、联盟或同等 SLG 功能证据的
  正式世界大地图入口；普通跨区开放地图使用 `map_entry`；`cg_end` 只记录开场 CG 结束并
  首次进入选角或可操作状态的时间，不记录中后段剧情演出结束。只有 `slg_entry` 截断体验趋势。
  `timestamp` 必须落在所属 `slice_index` 内。总览时间轴用高对比竖线、名称和精确 `M:SS`
  标出里程碑，点击后跳转到对应时间片。
- 时间片详情只展示截图、置信度、维度描述和待确认项；不展示推断、证据路径、
  `narrative_climax.reason` 或 `flow.reason`。
- 七维描述使用七维选项卡切换，空维度置灰；待确认项为空时隐藏。
- 高潮、低谷和心流高点仅作为总览时间轴的外部标识。
- 标题区与总览时间轴之间使用本地 SVG 绘制“全局情绪与体验曲线”，同图叠加情绪强度、
  体验实际值和体验趋势，三线可独立开关。
- 桌面曲线使用 `480～600px` 响应式图框，移动端保持 `320～420px`；按容器实际宽高重新计算
  SVG 坐标，禁止横向溢出，并为高分标记保留顶部安全区。
- 情绪节点按方向着色；剧情高潮使用菱形，心流高点使用星形。
- 每个时间片生成覆盖绘图区全高的透明命中带，相邻带以两个时间点横坐标中线为边界，必须连续、
  无缝且不重叠。带内任意高度悬停或聚焦时显示最终情绪分、剧情子分、其他刺激子分、drivers、
  主要刺激、评分原因、体验分、四项体验依据和回归预测值；点击后联动相同时间片详情。
- 时间片选中后不常驻显示命中框或引导线，只在悬停和键盘聚焦时显示临时高亮。桌面 Tooltip
  允许纵向超出曲线图框，画布不得裁切；移动端 Tooltip 固定在视口底部。
- 图表顶部显示回归起点、回归终点、差值及上升/持平/下降，图表下方提供“双曲线评分口径”。
- 图片区采用焦点轮播：当前图完整居中，前后图侧后方虚化；支持左右按钮、圆点、连续键盘切换、
  指针/触屏滑动和当前图放大，单图时隐藏导航。加载失败要显示可见错误，不能静默空白。
- 资源本地化，无 CDN；不得在 HTML 内复制一份独立分析对象。
- 直接用 `file://` 打开时若浏览器限制 `fetch`，使用页面内文件选择器加载该游戏对应的独立 JSON。

## 6. 飞书页签布局

- 只新建页签，不复用、不覆盖、不替换已有页签。同名时只允许用户提供新的唯一页签名或取消。
- 新页签第一行是合并总标题 `游戏前期体验拆解`。
- 第二行是时间轴；每个时间片一列，第一列为行标签。
- 后续严格七行：`阶段目标 (Experience Goal)`、`任务链 (Quest Chain)`、
  `核心循环 (Core Loop)`、`渐进体验预期 (New Content)`、
  `地图体验预期 (Map Progress)`、`经济体验预期 (Eco Progress)`、`剧情轴`。
- 连续同阶段的阶段目标单元格动态横向合并；所有范围从最终时间片数量计算，禁止固定旧行列。
- 飞书单元格只展示对应维度的 `fact`；不展示 `inference`、证据路径、置信度和内部索引。
- 无有效信息的单元格允许空置，不使用“未观察到”、`[Cont]` 或通用句强行填满。
- 剧情轴只展示剧情内容与叙事目的，不追加高潮/低谷/心流判断。
- 样式顺序：写值、合并、列宽、基础样式、局部样式、行高、回读、导出。
- Windows 下批量写值固定使用 `sheets +cells-set --cells @./<relative-json-file>`：
  单元格负载为 `{value: ...}` 对象，临时 JSON 必须创建在当前工作目录并在 `finally` 中删除。
  不得把含中文字符串的二维数组直接传给旧 `+write --values`。
- 使用主黄 `#FAC603`、浅黄 `#FAF1D1`、深灰 `#3A3E43`、白字 `#FFFFFF`；
  长文本插入换行并按最终内容动态计算行高。

## 7. 错误与停止条件

- `ffprobe`/`ffmpeg` 不存在或超时：停止，报告依赖或超时；不得生成虚构时间线/空图片。
- 视频不存在、时长非法或尾片异常：停止并修复输入；不手改时间片绕过生成器。
- JSON 语法错误、缺字段、多出第八维、边界/中点错误：以 `analysis_model.py` 报错为准，修复后重验。
- 图片缺失、为空、路径穿越、Windows 保留名或重名冲突：修复路径/文件后再构建。
- 同名页签：不写入；只接受新的唯一页签名后重新预检，或取消。
- `addSheet` 只发送标题；创建后按需调用独立的 `POST /insert_dimension_range` 扩展默认行列（不是 `sheets_batch_update`）。任一后续步骤失败时删除本次新建页签，清理失败必须进入错误 JSON。
- 权限错误：按 `lark-shared` 判断 user/bot；不要对 bot 运行用户登录。
- 页面值有但格式错：仍属失败，检查合并、样式、行高和导出结果。

## 8. 验证清单

本地：

- `analysis_model.py` 返回 0，规范化文件非空且 UTF-8 可读。
- 时间片从 0 连续覆盖到完整时长；2h17m 应为 44 片。
- 每片主图为中点；关键片证据图 1-3 张且都在片内。
- 七维齐全；所有结论能追到事实/证据，未知项已标记。
- `viewer/index.html` 可加载，所有时间片默认可见，详情、图片放大与错误提示可用。

飞书：

- `--dry-run` 前先执行 `sheets +info`/metadata 预检；可用 writer 的 `--preflight` 只读模式。
- 预检确认新页签名唯一后，`--dry-run` 计划正确；正式模式再次读取 metadata 检查冲突。
- 回读总标题、时间轴与七维行；读取页签元数据确认真实 `sheet_id` 与合并范围。
- 导出 xlsx，用 `openpyxl` 检查关键合并、标题/时间轴/维度行行高、标题底色/字色/字号、
  时间轴与维度列宽及长文本行高度。`openpyxl` 缺失即验证失败，不得跳过或记为通过。
- 行高验证对裁切风险保持严格：实际值低于目标超过 2pt 判失败；飞书导出量化造成的正向漂移
  允许最多 6pt，超过后仍判失败。
- 只有内容、元数据与导出样式三类检查都通过，才可报告完成。

复核后删除临时 xlsx。录屏、截图、临时 xlsx 与工作目录不进入版本控制。
