# 游戏前期体验拆解

## 核心原则

直接观看游戏录屏、截图及必要原视频片段，以同一份通过校验的 `analysis.json`
驱动本地 HTML 与飞书新页签。不能只 OCR，也不能只按分钟机械摘要；剧情与心流判断必须跨片聚合。

详细契约、七维口径、布局与错误处理见 [reference.md](reference.md)，执行前必须读完。

## 必读前置

开始前依次读取：

1. `rules/rules.md`。
2. `mistakes/` 中相关记录，至少包括图片理解、长图 OCR、飞书格式与输出格式类错题。
3. `spec/feishu-document-writing-special-flow-spec.md`。
4. `lark-shared` Skill。
5. `lark-sheets` Skill。

写飞书时遵循用户身份、最小权限、写前确认、回读与导出复核要求。

## 输入确认

开始处理前取得并复述：

- 录屏路径。
- 游戏名与版本；未知时标为待确认，不猜测。
- 目标飞书电子表格 URL。
- 目标新页签名。

目标 URL 或页签名未明确时，可以完成本地分析，但不得正式写飞书。本 Skill 只新建页签，
不复用、不覆盖、不替换已有页签。发现同名页签必须暂停，只允许用户提供新的唯一页签名或取消；
不能静默覆盖，也不能擅自加时间戳。

## 固定分析规则

- 分析层必须生成密集审阅时间点：`0-10m` 每 1 秒、`10-20m` 每 5 秒、`20m+`
  每 10 秒。密集审阅用于发现剧情、系统和反馈事件，不能因最终页面按分钟展示而省略。
- 展示层继续使用左闭右开时间片：`0-30m` 每 1m，`30-60m` 每 5m，`1h+`
  每 10m；尾片保留实际时长。HTML 与曲线只展示该层，不渲染数千个分析点。
- 每片主截图默认取中点。若中点无信息，例如纯黑、纯白或白屏/迷雾过场，则改选片内距离中点最近的
  临近有效帧，并在 `main_frame.selection_reason` 写入 `midpoint_uninformative`；60 秒片最多偏移 10 秒，
  更长片最多偏移片长 10% 且不超过 30 秒，不能为画面更美观随意偏移。主图只负责展示，
  不能代替密集审阅或关键片段连续回看。
- 关键转折补 1-3 张证据图。触发项包括：任务/阶段切换、系统首次开放、地图区域变化、
  经济获得或消耗、剧情冲突/过场/对话、失败/等待/难度突变、高潮、低谷、心流高点。
- 必须查看全部主截图；截图不足以辨认动作、音频、字幕、过场、对话或上下文时，查看必要原视频片段。
  无需线性观看无信息片段，但不得跳过任何主截图。
- 恰好填写七维：阶段目标、任务链、核心循环、渐进体验、地图体验、经济体验、剧情轴。
- 飞书显示名固定为：`阶段目标 (Experience Goal)`、`任务链 (Quest Chain)`、
  `核心循环 (Core Loop)`、`渐进体验预期 (New Content)`、
  `地图体验预期 (Map Progress)`、`经济体验预期 (Eco Progress)`、`剧情轴`。
- `fact` 是飞书与 HTML 最终展示文案，必须短、可横向比较；`inference` 仅供内部分析，不在交付页面展示。
  没有有效信息时两项允许空置，不得用“未观察到”或通用解释填满表格。
- 剧情轴跨片聚合：剧情判断只用 `none/climax/low`，心流判断只用 `none/flow_peak`，并关联时间与片内图片证据。
- `stage_range` 使用稳定 `stage_id`；相邻同 ID 的名称和范围必须一致，各阶段连续衔接。
- 根节点必须包含 `global_curves`，曲线点与现有时间片一一对应；七维信息保留在详情中，
  但不参与曲线加权。
- 根节点必须包含 `global_loops`。以可观察的小闭环为基本单位：每次出现明确动机、连续行为、即时奖励，
  并产生下一目标或再次投入时，独立记为一个 `micro_loop`。一次性剧情、单独教学步骤或过渡不得强行循环化。
  每个小 LOOP 只能归属一个主要大 LOOP，其他系统关系通过跨环连线表达。另用显式 `loop_families`
  按玩法语义归类小 LOOP，每个 `micro_loop` 必须填写唯一 `loop_family_id`；禁止用标题相似度自动猜测类型。

### 全局情绪与体验曲线

- **情绪强度**是以剧情为主体的综合情绪唤起强度。每片分别填写 `0～5` 的
  `narrative_score` 与 `supporting_score`，最终分固定为
  `round(0.7 × narrative_score + 0.3 × max(narrative_score, supporting_score), 1)`。
  纯环境/系统刺激最高只贡献 1.5 分，剧情 5 分仍保持最终 5 分。
- `drivers` 可多选 `narrative`、`environment_pressure`、`urgency`、`combat`、
  `progression_reward`、`relief`。其他刺激必须检查环境压力、暴风雪倒计时、紧迫目标、
  战斗冲突、成长奖励与危机缓解；有可观察刺激时不得因“没有剧情”直接记为 0。
- 另记 `positive/negative/mixed/neutral` 情绪方向、主要刺激和评分原因。负向高峰不代表剧情质量差。
- **体验强度**由分析者按 `0～5` 评分，用于判断玩法沉浸和心流强度；每片必须记录
  `gameplay_concentration`、`feedback_density`、`goal_challenge`、`interruption`
  四项文字依据与一句总评。
- 体验实际折线保留全部时间片人工评分；体验整体趋势先按 `stage_range.stage_id` 计算阶段平均分，
  再让每个阶段以相同权重做真实时间线性回归，避免长阶段因时间片更多而压倒关键体验跃迁。
  回归范围从录屏开始到首次 `slg_entry` 里程碑，包含入口所在时间片；入口后的实际折线继续显示，
  但不参与回归，趋势线也不延伸到入口之后。
  回归起点与终点预测差 `>=0.5` 为上升、`<=-0.5` 为下降，其余为持平；预测显示截断在
  `0～5`，且不修改原始评分，不得强制归一化为上升。
- 无法可靠判断时回看原视频；仍无法判断则降低置信度并写入待确认，禁止插值或套模板补分。

### 飞书七维口径

- **阶段目标**：`P序号. 阶段名 (English)` + 一句该阶段体验目标；连续时间片合并。
- **任务链**：只描述该时间片玩家的主要任务，使用 `Intro/Tutorial/Main/Unlock` 前缀；不是逐步操作日志。
- **核心循环**：描述系统对玩家的教学，以及当前教学到哪个循环环节；使用
  `[New]/[Upgrade]/[Loop]/[Cont]/[End] + T编号 + 系统 + (环节)`。
- **渐进体验预期**：只写期间解锁了什么系统、机制或界面；没有新解锁则留空。
- **地图体验预期**：只写期间解锁了哪些地图或区域；没有地图变化则留空。地图层级按功能边界判断，
  不能仅凭镜头远近或“画面里出现建筑”判断。
- **经济体验预期**：只写玩家等级、战力值、主城等级及用户指定的核心建筑等级等成长快照；
  例如寒霜记录大熔炉等级。只记录可辨识的实际等级，升级条件不等于当前等级。
- **剧情轴**：只写这段剧情在讲什么，以及叙事目的；没有剧情推进则留空。死亡、离别、背叛、
  牺牲、重大胜利和继承目标等关键过场必须查看完整片段与连续字幕，不能用分钟中点图代替。
- 背景设定卡、档案、旁白式世界观文本只能记录为“设定信息”；除非连续画面、对话和结果明确证明
  事件在片内实际发生，否则不得改写成实际剧情，不得仅凭立绘、黑底文字或人物姿态判定剧情高潮。

飞书与 HTML 不展示“事实：”“推断：”前缀，也不追加值为 `none` 的高潮、低谷或心流说明。

### HTML 展示口径

- 使用单一总览时间轴代替七条泳道，时间片按真实时间比例压缩到容器宽度，不产生页面横向滚动。
- 点击时间片后，在详情区通过七维选项卡切换 `fact` 描述；七维始终保留，空维度置灰。
- 详情只展示截图、置信度、维度描述和待确认项；不展示推断、证据路径及内部聚合原因。
- 待确认项为空时隐藏；高潮、低谷和心流高点只作为总览时间轴上的外部标识。
- 截图使用焦点轮播：当前图完整居中，前后图侧后方虚化；支持左右按钮、圆点、连续键盘切换、
  触屏滑动和当前图放大，单图时隐藏导航。
- 标题区与总览时间轴之间增加本地 SVG“全局情绪与体验曲线”：同图叠加情绪强度、体验实际值
  和体验趋势，支持三线独立开关。每片使用覆盖绘图区全高的透明命中带，带内任意高度均可
  悬停预览、聚焦和点击联动时间片详情；选中后不常驻显示框线，只在悬停或键盘聚焦时高亮。
- 桌面悬浮说明允许超出曲线图框显示，不得被画布裁切；移动端继续固定在视口底部。
- 桌面曲线使用高度 `480～600px` 的响应式图框，移动端保持 `320～420px`；按容器实际宽高重绘，
  禁止横向溢出，并为高潮菱形与心流星标保留顶部安全区。
- 剧情高潮用菱形标记，心流高点用星形标记；图表顶部展示回归起点、回归终点、差值及
  上升/持平/下降结论，曲线下方提供可折叠“双曲线评分口径”。
- 精确转折写入根节点 `timeline_milestones`：传统 SLG 世界大地图入口使用 `slg_entry`，
  普通开放区/跨区地图入口使用 `map_entry`，明确的无 HUD 过场结束并恢复可操作状态使用 `cg_end`。
  总览时间轴必须显示带名称和 `M:SS` 时间的竖向标记，并可点击跳转到所属时间片。
  只有 `slg_entry` 截断体验趋势；`map_entry` 与 `cg_end` 均不得截断。前置地图/雷达、加载界面、
  普通地图入口与正式 SLG 入口必须分开记录。
- 时间片详情之后、页面最后固定增加“全局 LOOP 流程”。所有小 LOOP 按实际时间组成一条阅读主链，
  每个小 LOOP 直接展开“动机 → 行为 → 奖励 → 下一动机”四节点；不得再把整套系统画成单张巨型四格卡。
- 大 LOOP 仅用于分组小 LOOP：聚落建设为青蓝、战斗远征为橙红、英雄养成为紫色。连续同类小 LOOP
  使用同色分段背景带包裹；被其他流程打断后再次出现时，使用同色 `macro_return` 回流线连接分段。
  不同大 LOOP 的供给关系使用 `cross_macro` 从具体奖励/下一动机连向目标动机。
- 大 LOOP 图例只淡化对应分段，不删除或重排时间主链。点击小 LOOP 联动首个可见关联时间片；
  LOOP 区不显示悬浮窗，四段内容直接常驻卡片，`evidence_frames` 只用于校验和追溯。
- 全局 LOOP 标题下展示事件级小 LOOP 数、去重类型数，以及各类型的出现次数和强化次数；
  `强化次数 = max(出现次数 - 1, 0)`。时间片详情显示全部关联小 LOOP 标签，点击后滚动至对应卡片并短暂高亮；
  没有关联时明确显示“当前时间片无关联 LOOP”。
- 主体流程终点由项目证据确定。寒霜以 P8 完整主城解锁为终点；P9 进入 SLG 大地图只作为图外虚线出口。

## 执行顺序

所有命令从本 Skill 目录执行；路径含空格时加引号。

1. 生成展示时间片与密集审阅时间点：

   ```powershell
   py -3 scripts/video_timeline.py "<video>" --output "<work>/timeline.json"
   ```

2. 首轮按中点抽展示帧；同时按 `review_points` 生成审阅索引或接触表：

   ```powershell
   py -3 scripts/extract_frames.py "<video>" --output-dir "<work>/frames" --mapping-output "<work>/frames.json"
   ```

3. 人工或具备视觉能力的模型观看全部密集审阅帧、展示主图与必要原视频，记录关键转折。
   发现对话、CG、倒地、死亡、离别、牺牲、身份揭示、重大胜利或章节转场时，必须连续查看
   候选点前后原视频，并以 0.5～1 秒精度确定事件。需要补图时生成
   `evidence-times.json`，格式为 `{"evidence_times":{"<slice_index>":[<absolute_seconds>]}}`，
   再用 `--evidence-config` 重跑 `extract_frames.py`。依据 [reference.md](reference.md)
   形成 `<work>/analysis.json`，并为每个现有时间片补齐情绪强度、体验强度及结构化评分依据。

4. 校验并规范化唯一事实源：

   ```powershell
   py -3 scripts/analysis_model.py "<work>/analysis.json" --output "<work>/analysis.validated.json"
   ```

   后续只使用该校验结果，并将其作为最终 `analysis.json`。

5. 从已验证时间片聚合 `global_loops`：先按“明确动机、连续行为、即时奖励、下一目标”拆成小 LOOP，
   再为每个小 LOOP 指定唯一主要大 LOOP和显式语义类型，并按真实时间建立主链、`macro_return` 与 `cross_macro`。
   每个节点引用已有时间片与图片证据，无法确认的门槛标为 `pending_confirmation`。

6. 构建固定本地包：

   ```powershell
   py -3 scripts/build_viewer.py "<work>/analysis.json" --output-dir "<output>/viewer" --dataset-id "<game-slug>" --dataset-name "<显示名>"
   ```

   多个拆解默认共用一个查看器，固定交付为 `viewer/index.html`、
   `viewer/data/datasets.json`、`viewer/data/<game-slug>.json`、
   `viewer/screenshots/<game-slug>/`。构建时用 `--dataset-name "<显示名>"` 写入清单。
   每个拆解只能读取自己的 JSON 和截图目录，不得把不同游戏写入同一个数据文件。
   标题区“拆解项目”下拉框读取清单并通过 `?dataset=<game-slug>` 切换；无参数时读取构建时指定的数据文件。
   同一输出目录再次用新的 `--dataset-id` 构建时必须保留已有数据集，只替换当前同名数据集。
   若通过 `file://` 打开导致浏览器限制 `fetch`，使用页面文件选择器加载对应的独立 JSON。

7. `dry-run` 前必须通过 `sheets +info`/metadata 只读预检目标和同名冲突。使用脚本预检：

   ```powershell
   py -3 scripts/write_feishu_sheet.py "<work>/analysis.json" "<sheet_url>" "<new_tab_name>" --identity user --preflight
   ```

   同名时只能取得新的唯一页签名后重新预检，或取消。不得复用已有页签，也不得替换或覆盖。

8. 预检通过后再预演：

   ```powershell
   py -3 scripts/write_feishu_sheet.py "<work>/analysis.json" "<sheet_url>" "<new_tab_name>" --identity user --dry-run
   ```

9. 仅当用户目标已明确、目标元数据已核对且无未决同名冲突时正式写入。正式模式会再次
   执行 `sheets +info`/metadata 检查，防止预检后出现同名页签：

   ```powershell
   py -3 scripts/write_feishu_sheet.py "<work>/analysis.json" "<sheet_url>" "<new_tab_name>" --identity user
   ```

10. 回读关键范围与页签元数据，并导出 xlsx 复核标题、七维行、动态合并、列宽、行高、
   字号、底色、字色和长文本可见性。

## 完成门槛

只有同时满足以下条件才能声称完成：

- `analysis_model.py` 校验通过；时间片连续覆盖全视频，主图、证据图存在且可打开。
- 已查看全部主截图及必要原视频片段；无需线性观看无信息片段。
- 已抽查共享 `viewer/index.html` 能通过标题区下拉框切换全部清单数据集，页面不存在顶部筛选栏，
  每个数据集只能加载自己的 JSON 和截图；全局情绪与体验曲线、趋势摘要、图例开关、
  全高命中带联动、时间轴、阶段快速跳转、七维、证据图、详情关联 LOOP 跳转、LOOP 去重统计与页面末尾流程可用；
  LOOP 区不存在悬浮窗。
- 飞书新页签标题正确，七维及剧情标记完整；内容回读、合并元数据检查、xlsx 导出样式复核均通过。
- 新页签创建后任一步失败均已删除本次新建页签；若清理失败，错误 JSON 明确列出 cleanup 状态。
- HTML 与飞书内容可追溯到同一 `analysis.json`。

不允许仅凭 API success、退出码 0 或单元格有值声称完成。

## 失败处理

- 缺少 `ffmpeg`/`ffprobe`、视频不存在、非法 JSON、截图缺失、校验失败：停止后续步骤，报告原始错误与修复方式。
- 画面、音频或数值不可辨识：降低置信度并写入待确认；需要时请求高清录屏或用户确认，禁止补脑。
- 同名页签：停止写入；只接受新的唯一页签名后重试，或取消。
- 权限/认证失败：按 `lark-shared` 区分 user 与 bot，最小范围授权后重试。
- 写入、回读、合并或导出任一失败：交付状态为未完成；best-effort 删除本次新建页签并报告清理结果，避免残页阻塞重试。

## 仓库卫生

不提交录屏、截图、临时 xlsx、授权信息或工作目录产物。只保留用户要求的最终交付；
验证用导出文件在复核后删除。
