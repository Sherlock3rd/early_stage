# 游戏前期体验拆解：情绪与体验双曲线设计

## 背景与问题

现有“综合情绪曲线”把七维对玩家情绪的正负贡献进行加权。该结果同时混入剧情情绪、
任务推进、玩法反馈、地图和成长信息，不能直接回答以下问题：

1. 玩家在哪些时间点产生最强烈的剧情情绪？
2. 玩家在哪些时间点最沉浸、最接近心流？
3. 前期玩法体验是否随时间形成整体上升趋势？

因此移除七维加权综合线，改为两条语义独立的人工评分曲线。七维拆解继续保留在时间片详情中，
但不再参与曲线计算。

## 目标

- 用“情绪强度”定位剧情情绪高点，不把悲剧情节错误画成低谷。
- 用“体验强度”定位玩法沉浸和心流高点。
- 用全部体验评分点的线性回归直线展示整体变化趋势，不强制把真实数据修正为上升。
- 让每个分数都能通过录屏事实和结构化评分依据解释。
- 在同一张图中对齐两条曲线，便于观察剧情刺激与玩法沉浸是否同步。

## 非目标

- 不把曲线分数解释为游戏整体质量评分。
- 不使用玩家生理数据、问卷或真实遥测；当前数据仍是基于录屏证据的分析判断。
- 不强制归一化为上升趋势；没有上升时应如实显示。
- 本轮不修改飞书页签。

## 时间颗粒度

评分点与现有体验时间片一一对应：

- `0–30min`：每 1 分钟一个点。
- `30–60min`：每 5 分钟一个点。
- `1h+`：每 10 分钟一个点。
- 尾片使用视频实际结束时间。
- 不通过插值伪造更细颗粒度。

## 曲线一：剧情情绪强度

### 含义

表示当前剧情事件带给玩家的情绪刺激有多强，不表达正负好坏。

### 评分

- `0`：没有明显剧情或情绪刺激。
- `1`：轻微氛围、期待或关系铺垫。
- `2`：可感知的剧情推进，但情绪刺激有限。
- `3`：明确的冲突、胜利、失落或重要揭示。
- `4`：强烈转折或关键角色事件。
- `5`：核心剧情高潮或全段最强情绪事件。

每个点还必须记录：

- `valence`：`positive`、`negative`、`mixed`、`neutral`。
- `event`：可观察到的剧情事件。
- `reason`：该事件为何对应当前强度。

例如“比尔去世”应记录为 `intensity: 5`、`valence: negative`，并在图中显示为高峰。
负向颜色只表示情绪方向，不代表剧情质量差。

## 曲线二：玩法体验/心流强度

### 含义

表示玩家在当前时间片中的玩法投入、反馈连续性和沉浸程度，用于辅助判断心流强度。

### 评分

- `0`：无有效玩法或完全停滞。
- `1`：明显等待、重复菜单或持续打断。
- `2`：玩法碎片化，存在目标但反馈稀疏。
- `3`：目标明确、操作与反馈稳定，可持续投入。
- `4`：挑战、操作和反馈连续，形成强心流。
- `5`：全段玩法沉浸峰值。

总分由分析者综合判断，不使用机械加权。每个点必须记录四项文字依据：

1. `gameplay_concentration`：有效玩法操作在该时间片中的浓度。
2. `feedback_density`：操作、战斗、奖励、成长等反馈的频率与清晰度。
3. `goal_challenge`：目标是否明确，挑战是否与当前能力匹配。
4. `interruption`：菜单、过场、等待、重复教学或信息过载造成的打断。

同时记录一句 `summary`，说明为何得到当前体验分。

## 数据契约

移除根字段 `emotion_curve`，替换为：

```json
{
  "global_curves": {
    "scale": {"min": 0, "max": 5},
    "points": [
      {
        "start": 120,
        "end": 180,
        "emotion": {
          "intensity": 5,
          "valence": "negative",
          "event": "比尔去世，众人在墓前送别",
          "reason": "核心亲人离世并触发遗志继承"
        },
        "experience": {
          "score": 2,
          "basis": {
            "gameplay_concentration": "以过场和任务承接为主，主动操作较少",
            "feedback_density": "获得遗志与新任务反馈",
            "goal_challenge": "长期建设目标变得明确",
            "interruption": "剧情过场中断玩法操作"
          },
          "summary": "剧情推动强，但玩法沉浸强度较低"
        }
      }
    ]
  }
}
```

### 校验

- `global_curves`、`scale`、`points` 必填。
- `scale` 固定为 `0～5`。
- `points` 数量、顺序、`start`、`end` 必须与 `slices` 完全一致。
- `emotion.intensity` 和 `experience.score` 必须是 `0～5` 的有限数。
- `emotion.valence` 只允许四个规定枚举。
- `emotion.intensity > 0` 时，`emotion.event` 和 `emotion.reason` 必须为非空文本；
  强度为 0 时允许二者为空，避免填充“无明显事件”等无效文字。
- 四项 `experience.basis` 和 `experience.summary` 必须为非空文本，用于解释人工体验总分。
- 不再要求七维情绪权重，也不再计算七维综合值。

## 体验趋势算法

只对体验实际分计算普通最小二乘线性回归：

- 每个观测点的 `x` 为时间片真实起止时间的中点，单位为秒。
- 每个观测点的 `y` 为人工填写的 `experience.score`。
- 使用全部观测点拟合 `y = slope × x + intercept`。
- 只有一个观测点，或全部观测点时间相同时，回归线为该体验分对应的水平线。
- 回归线从首个评分点的时间中点绘制到最后一个评分点的时间中点。
- 回归预测值超出 `0～5` 时，仅在显示和摘要中截断到该量表范围；原始体验分不变。

整体趋势比较：

- `opening_prediction`：回归线在首个评分点的预测值。
- `ending_prediction`：回归线在最后一个评分点的预测值。
- `delta = ending_prediction - opening_prediction`。
- `delta >= 0.5`：上升。
- `delta <= -0.5`：下降。
- 其他：持平。

趋势结果只描述录屏中的实际变化，不修改原始分数。

## HTML 信息结构

保留一个“全局曲线”区块，在同一张 SVG 图中叠加：

1. 情绪强度线：橙色主线；节点按情绪方向着色。
2. 体验实际线：蓝色细线。
3. 体验整体趋势线：青色粗直线。

三条线共用 `0～5` 纵轴和真实视频时间横轴。

### 标记

- `narrative_climax=climax` 使用菱形剧情高潮标记。
- `flow=flow_peak` 使用星形心流高点标记。
- 标记与分数独立，不能仅凭分数自动生成现有判断字段。

### 图例与交互

- 图例可分别开关“情绪强度”“体验实际值”“体验趋势”。
- 提供“显示全部”。
- 悬停或聚焦同一时间点时，显示：
  - 时间范围；
  - 情绪强度、方向、事件和原因；
  - 体验分、四项依据和总评；
  - 该时间点对应的线性回归预测值。
- 点击节点联动下方对应时间片详情。
- Escape 关闭提示。
- 筛选隐藏的时间片节点降低透明度并禁止跳转。

### 趋势摘要

图表顶部显示：

- 回归起点预测值；
- 回归终点预测值；
- 差值；
- “上升 / 持平 / 下降”结论。

同时提供一句阅读说明：

> 情绪曲线用于定位剧情情绪高点；体验曲线用于判断玩法沉浸与心流强度；趋势线用于观察前期体验是否整体提升。

## 数据流

1. 录屏分析阶段继续生成七维时间片事实。
2. 分析者为每个时间片人工填写情绪强度和体验强度及依据。
3. `analysis_model.py` 校验双曲线数据与时间片边界。
4. `build_viewer.py` 原样复制校验后的统一数据。
5. `viewer.js` 以真实时间中点计算体验线性回归和整体趋势，只负责派生展示，不修改原始分数。
6. 图表、总览时间轴和详情区共享同一选中时间片状态。

## 响应式图框

- 曲线图容器高度使用 `clamp(320px, 30vw, 420px)`，避免宽屏下按原始宽高比持续增高。
- SVG 宽高始终匹配容器实际尺寸；渲染模型以容器像素尺寸重算坐标，不拉伸文字和标记。
- 监听容器尺寸变化并重新绘制，保证窗口缩放和窄屏下曲线、坐标轴及标记保持在边框内。
- 图框不产生横向滚动；移动端提示层仍固定在视口范围内。

## 错误处理

- 双曲线缺失、分数越界、枚举错误、依据为空或时间边界不一致时停止构建。
- 图表渲染错误时显示可见提示，但不阻断时间片详情和截图查看。
- 无法可靠评分时不得套用固定模板；应回看录屏，仍无法判断则在原时间片加入待确认项。

## 测试与验收

自动化测试覆盖：

- 双曲线必填字段、分数范围、枚举和时间片对齐校验。
- 单点、双点和多点的线性回归；覆盖水平、上升、下降和预测值截断。
- 上升、持平和下降阈值。
- 三条线、情绪方向颜色、剧情高潮和心流标记。
- 桌面与窄屏图框高度、SVG 实际尺寸同步和尺寸变化重绘。
- 图例开关、提示内容转义、Escape 关闭和节点联动。
- 旧有筛选、时间轴、七维详情和图片轮播无回归。
- 输出中不再出现七维权重、综合情绪值或旧加权算法说明。

寒霜示例验收：

- 比尔去世是负向情绪高峰。
- 首次出征和连续战斗是体验高点。
- 连续经营菜单是体验低谷。
- 完整主城与 SLG 大地图解锁带来后段体验回升。
- 图表明确展示实际体验是否形成整体上升趋势。
