# 游戏前期体验全局 LOOP 流程设计

## 1. 目标

在游戏前期体验拆解 HTML 的最后新增“全局 LOOP 流程”部分。流程以玩家的动机、行为、奖励和
下一动机为基本语义，帮助策划识别前期体验中真正形成闭环的玩法，以及各闭环之间如何相互供给。

寒霜示例的主体范围为 `0:00–25:00`，结束于 P8 完整主城解锁。P9“编队并进入 SLG 大地图”
只作为主体外的虚线出口提示，不纳入 LOOP 分析。

本轮不修改飞书页签。

## 2. 设计原则

### 2.1 LOOP 判定

只有形成以下完整结构的玩家体验才是独立 LOOP：

`动机 → 行为 → 奖励 → 下一动机或再次投入`

系统教学步骤是 LOOP 内的行为节点，不因菜单或系统名称不同而机械拆分。只发生一次、没有回流的
剧情、任务触发、法令选择、地图揭示等内容使用线性节点表达。

### 2.2 自然分段

章节仅用于改善阅读，不预设数量。分析者根据实际流程转折决定是否分章；没有必要时允许不设置章节。
不得为了贴合参考图中的 CP 数量强行合并或拆分游戏阶段。

### 2.3 证据约束

流程结论必须来自已经查看的录屏、主图、证据图和通过校验的七维数据。参考图只提供布局和颜色语义，
不能作为寒霜玩法内容的证据。无法确认的 SLG 门槛、活动入口或系统因果必须明确标为待确认。

## 3. 寒霜流程结构

### 3.1 线性入口

1. 暴风雪中的生存压力驱动玩家寻找落脚点、点燃取暖设施并接触幸存者。
2. 比尔去世与避难所蓝图完成叙事转折，把短期求生目标转换为建设聚落的长期动机。

以上内容属于全局流程入口，不单独包装成可重复 LOOP。

### 3.2 聚落建设 LOOP

- 动机：延续比尔遗志，应对暴风雪、低温和设施等级门槛，并为出征做准备。
- 行为：建造与升级、配置建造队列、派遣幸存者、生产和提交订单。
- 奖励：取暖与安全、资源产出、大熔炉等级、战力提升、新建筑和功能开放。
- 下一动机：更高等级需求、资源缺口、瞭望塔提示和新的出征准备推动玩家再次投入建设。

### 3.3 战斗远征 LOOP

- 动机：清除外部威胁、获取聚落发展物资并扩大探索范围。
- 行为：编队出征、小队战斗、载具远征和连续战斗。
- 奖励：战利品、战力提升、探索推进和英雄相关内容。
- 下一动机：战利回投聚落建设，或由更强敌人驱动队伍养成与下一次远征。

### 3.4 英雄养成 LOOP

- 动机：应对更强敌人并提升远征效率。
- 行为：招募、编队、养成以及背包和奖励整理。
- 奖励：新英雄、队伍强化、技能与宝箱反馈。
- 下一动机：强化结果重新投入战斗远征。

### 3.5 跨 LOOP 关系

- 建设奖励支撑远征。
- 远征奖励回投建设。
- 远征压力触发英雄养成。
- 英雄养成奖励回到远征。

跨 LOOP 回流箭头表达复合循环，不重复建立同义 LOOP。

### 3.6 线性出口

极寒法令和设施升级作为出口前的线性过渡。P8 完整主城解锁是主体终点；终点后使用虚线箭头指向
“进入 SLG 大地图”，并标明该节点不属于本图分析范围。

## 4. 数据契约

`analysis.json` 根节点新增必填对象 `global_loops`：

```json
{
  "global_loops": {
    "scope": {
      "start": 0,
      "end": 1500,
      "end_label": "完整主城解锁",
      "outside_exit_label": "进入 SLG 大地图"
    },
    "chapters": [],
    "nodes": [],
    "edges": []
  }
}
```

### 4.1 `scope`

- `start`、`end`：主体流程时间边界，必须位于视频时长内且 `start < end`。
- `end_label`：主体终点名称。
- `outside_exit_label`：主体外出口名称，仅作虚线提示。

### 4.2 `chapters`

可为空。每项包含唯一 `id`、显示名 `title`、`start` 和 `end`。章节必须位于 `scope` 内，
互不重叠并按时间排序。节点可通过 `chapter_id` 归组；没有章节时该字段为空。

### 4.3 `nodes`

公共字段：

- `id`：全局唯一稳定 ID。
- `type`：`entry`、`transition`、`loop`、`end` 或 `outside_exit`。
- `title`：节点标题。
- `summary`：简短说明。
- `chapter_id`：可为空；非空时必须引用现有章节。
- `slice_indices`：关联时间片索引，至少一个；`outside_exit` 可引用边界后的首个时间片。
- `evidence_frames`：引用关联时间片已有主图或证据图。
- `status`：`confirmed` 或 `pending_confirmation`。

`loop` 节点额外必填：

- `motivation`：玩家为何投入。
- `behaviors`：非空行为数组，记录闭环中的核心行为，不记录逐步点击日志。
- `reward`：玩家获得的反馈、资源、能力或进度。
- `next_motivation`：奖励或压力如何推动下一轮。
- `accent`：`settlement`、`expedition` 或 `hero_growth`，用于寒霜三个 LOOP 的稳定配色。

非 `loop` 节点不得伪造上述闭环字段。

### 4.4 `edges`

- `from`、`to`：必须引用现有节点。
- `kind`：`primary`、`feedback` 或 `conditional`。
- `label`：连线说明。
- `primary` 表示线性主推进；`feedback` 表示 LOOP 回流；`conditional` 表示待确认或条件触发。

入口到主体终点必须存在一条 `primary` 路径。允许 `feedback` 成环，但不得存在无入口的孤立节点。
`outside_exit` 只能位于主体终点之后，并通过 `conditional` 虚线边连接。

## 5. HTML 布局

页面顺序固定为：

1. 全局情绪与体验曲线
2. 体验总览时间轴
3. 时间片详情
4. 全局 LOOP 流程

全局 LOOP 流程始终位于详情之后，是页面最后一个主要部分。

### 5.1 总体结构

- 使用纵向主推进结构。
- 线性入口位于顶部。
- 中部为三个 LOOP 组成的循环生态；跨 LOOP 箭头展示供给和回流。
- 线性出口与主体终点位于底部。
- SLG 大地图入口位于主体框外，以虚线连接。

### 5.2 色块语义

LOOP 容器主色：

- 聚落建设：青蓝。
- 战斗远征：橙红。
- 英雄养成：紫色。

LOOP 内角色色固定：

- 动机：黄色。
- 行为：紫蓝色。
- 奖励：绿色。
- 下一动机：黄色虚线框。

其他节点：

- 关键剧情触发：红色。
- 普通线性过渡：灰蓝色。
- 主体终点：金色。
- 主体外出口：低透明度虚线框。

颜色不是唯一识别手段；所有色块同时展示明确文字标签。

### 5.3 响应式规则

- 桌面端使用全宽布局，LOOP 内按“动机 → 行为 → 奖励 → 下一动机”横向排列。
- 窄屏改为纵向卡片，箭头方向同步转为自上而下。
- 页面不得产生横向滚动；文本允许换行，不截断关键说明。
- 连线使用本地 SVG，根据实际容器尺寸和节点位置重算；窗口尺寸变化时重新渲染。

## 6. 交互

- 节点悬停或键盘聚焦时展示关联时间、证据截图和简短依据。
- 点击节点后选择其主关联时间片，并滚动到时间片详情。
- 若节点全部关联时间片被当前筛选隐藏，节点置灰并设置禁用语义，不跳转，也不自动清空筛选。
- LOOP 卡片具有明确可聚焦区域，支持 Enter 和 Space。
- Tooltip 可越出流程图卡片显示，不能裁切长文本；移动端使用视口内固定或受限布局。
- 分析文本、标题、标签和证据说明全部执行 HTML 转义。

## 7. 校验与错误处理

校验器必须拒绝：

- `scope` 越过视频范围或起止无效。
- 主体节点引用 `scope.end` 之后的时间片。
- LOOP 缺少动机、行为、奖励或下一动机。
- 节点 ID 重复、章节引用失效、连线悬空。
- 证据不属于节点关联时间片。
- 无入口到终点的主路径。
- 存在无法从入口到达的孤立主体节点。
- `outside_exit` 被当作主体 LOOP，或使用实线主推进边连接。

Viewer 遇到流程数据错误时，在最后一部分显示明确错误信息；情绪曲线、时间轴和详情仍可继续使用。

## 8. 验收

### 8.1 自动化测试

- `global_loops` 有效数据通过，缺字段和错误引用被拒绝。
- 寒霜包含三个独立 LOOP，并存在建设与远征、养成与远征的回流关系。
- 入口、过渡、终点和主体外出口按正确类型渲染。
- 三个 LOOP 容器与四类内部角色具有稳定类名和颜色语义。
- 所有用户数据均被转义。
- 筛选禁用、点击跳转、键盘操作和 ResizeObserver 重绘无回归。
- 移动端切换为纵向布局且页面无横向溢出。

### 8.2 浏览器验收

桌面和移动端均需确认：

- 三个 LOOP 边界清晰，动机、行为、奖励和下一动机无需猜测。
- 跨 LOOP 主推进与回流方向正确，箭头不遮挡文字。
- 长文本、Tooltip 和证据预览不被裁切。
- 点击节点跳转到正确时间片详情。
- P8 是主体终点，SLG 大地图仅显示为图外虚线出口。
