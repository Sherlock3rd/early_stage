# 游戏前期体验拆解：全局情绪曲线设计

## 目标

在现有游戏前期体验 HTML 查看器中新增“全局情绪曲线”区块。曲线按已有时间片展示七个分析维度对玩家当下情绪的正负贡献，并生成一条可解释、可复算的综合情绪曲线。

参考表为“比格FTE叙事拆环”的“图表”与“拆环”页签。参考算法使用显式权重：

`体验曲线 = 1.5 × 近 3 节点情绪值标准差 + 1.0 × 玩法浓度`

本功能保留“分项评分 + 显式权重 + 综合曲线”的方法，但不直接复制参考表的两个指标。新曲线使用当前体验拆解固定七维，并保留正负情绪方向。

## 时间颗粒度

- 曲线点与现有体验时间片一一对应。
- `0–30min`：每 1 分钟一个评分点。
- `30–60min`：每 5 分钟一个评分点。
- `1h+`：每 10 分钟一个评分点。
- 最后一个评分点覆盖视频实际尾片，不补齐到完整区间。
- 每个评分点代表整个时间片，不通过插值伪造分钟级数据。

## 七维评分

每个时间片的七个维度分别评分，范围为 `-5～+5`：

- `-5`：该维度造成强烈负向情绪。
- `-3`：造成明显负向情绪。
- `0`：中性、没有明显情绪影响，或该片没有该维度事件。
- `+3`：造成明显正向情绪。
- `+5`：造成强烈正向情绪高点。

维度评分表达玩家当下的情绪效价，不表达设计质量。例如角色死亡可以在剧情轴得到负分，但不代表剧情质量差。

维度判断口径：

1. 阶段目标：目标是否清晰、推进是否可感知、阶段收束是否带来满足；迷失或目标中断为负向。
2. 任务链：任务指引、行动路径和完成反馈是否顺畅；卡顿、反复或不知下一步为负向。
3. 核心循环：挑战、操作、即时反馈和能力匹配带来的情绪；受控挑战可正向，失控或无反馈可负向。
4. 渐进体验：新系统、新机制或新界面带来的新鲜感；信息过载、强制教学堆叠可负向。
5. 地图体验：新区域发现、空间展开和探索期待；受限、迷路或无效移动可负向。
6. 经济体验：等级、战力、主城或核心建筑成长带来的获得感；资源卡点、成长停滞或损失可负向。
7. 剧情轴：剧情事件带来的即时情绪方向与强度；悲伤、恐惧、绝望为负，胜利、希望、团聚为正。

每个评分点必须附一条简短摘要，说明本片综合情绪变化的主要原因。无法可靠判断时不得补脑，应降低原时间片置信度并加入待确认项。

## 默认权重与公式

默认权重：

- 阶段目标：`0.10`
- 任务链：`0.15`
- 核心循环：`0.20`
- 渐进体验：`0.10`
- 地图体验：`0.10`
- 经济体验：`0.15`
- 剧情轴：`0.20`

权重总和固定为 `1.00`。综合情绪值按下式计算：

`E(t) = Σ(score_dimension(t) × weight_dimension)`

因为各维度分数范围为 `-5～+5` 且权重和为 `1`，综合值也保持在 `-5～+5`。

默认权重写入 `analysis.json`，允许项目级覆盖。覆盖后仍要求所有七维权重大于等于 0 且总和为 1。HTML 从数据中的权重实时计算综合值，不额外维护第二份综合分数。

## 数据结构

`analysis.json` 根节点新增必填字段：

```json
{
  "emotion_curve": {
    "scale": {"min": -5, "max": 5, "neutral": 0},
    "weights": {
      "阶段目标": 0.1,
      "任务链": 0.15,
      "核心循环": 0.2,
      "渐进体验": 0.1,
      "地图体验": 0.1,
      "经济体验": 0.15,
      "剧情轴": 0.2
    },
    "points": [
      {
        "start": 0,
        "end": 60,
        "scores": {
          "阶段目标": 2,
          "任务链": 1,
          "核心循环": 1,
          "渐进体验": 0,
          "地图体验": 2,
          "经济体验": 0,
          "剧情轴": 1
        },
        "summary": "进入新环境并建立生存目标，整体保持轻度正向期待。"
      }
    ]
  }
}
```

校验规则：

- `emotion_curve`、`scale`、`weights`、`points` 必填。
- `weights` 与 `scores` 必须恰好包含固定七维。
- 所有分数必须是 `-5～+5` 的有限数值。
- 权重必须是有限非负数，误差范围内总和为 `1`。
- `points` 数量、顺序、`start` 和 `end` 必须与 `slices` 完全一致。
- `summary` 必须是非空简短文本。

## HTML 布局与交互

“全局情绪曲线”位于筛选区之后、体验总览时间轴之前。

- 使用本地 SVG 绘制，不引入 CDN 或外部图表库。
- 横轴为真实视频时间，纵轴固定 `-5～+5`。
- `0` 轴使用明显分隔线，正负区域使用低透明度背景区分。
- 综合曲线默认高亮加粗。
- 七维曲线默认低透明度显示，各自使用稳定颜色。
- 图例支持逐维开关、高亮，以及“只看综合”“显示全部”。
- 悬停节点显示时间范围、七维分数、综合值和评分摘要。
- 点击曲线节点后，联动选中下方相同时间片并滚动到详情区。
- 现有剧情高潮、剧情低谷和心流高点标记继续保留在总览时间轴，不混入综合情绪公式。
- 图表下方提供可折叠“评分与加权算法”，显示当前权重、分数范围和公式。
- 小屏设备保持纵轴与综合曲线可读；图例可横向滚动，但页面本身不产生横向滚动。

## 数据流

1. 录屏分析阶段对每个现有时间片完成七维事实与七维情绪评分。
2. `analysis_model.py` 同时校验体验切片与情绪曲线。
3. `build_viewer.py` 继续只复制同一份校验后的 `analysis.json`，不生成第二套情绪数据。
4. `viewer.js` 根据 `weights` 和 `scores` 计算综合值并绘制 SVG。
5. HTML 图表与时间轴、详情区共享同一个选中时间片状态。

## 错误处理

- 缺失维度、分数越界、权重和不为 1、评分点与切片不对齐时，校验失败并停止构建。
- 评分数据无法判断时不允许用自动插值或固定模板填充。
- 图表渲染异常时显示可见错误，不影响用户继续查看已加载的时间片详情。

## 测试与验收

自动化测试覆盖：

- 权重、分数范围、七维完整性和时间边界校验。
- 综合值公式及自定义权重。
- SVG 坐标映射、零轴、正负点和单点尾片。
- 七维图例开关、“只看综合”“显示全部”。
- 悬停内容转义，避免分析文本注入 HTML。
- 曲线节点点击后联动正确时间片。
- 旧有筛选、时间轴、详情选项卡和截图轮播不回归。

实际页面验收：

- 综合曲线和七维分项均可辨识。
- 正负方向与关键剧情、任务和系统体验相符。
- 点击任一点可定位到对应时间片。
- 算法说明可查看当前实际权重与公式。
- 页面无外部网络依赖、无横向页面滚动。
