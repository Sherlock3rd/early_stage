# `game-early-experience-breakdown` 需求级会话

## 基本信息
- 工具名称：`game-early-experience-breakdown`
- 当前状态：寒霜、三冰与 Dark War 本地拆解已完成并接入统一查看器；三项飞书页签均已生成
- 责任视角：系统策划拆解游戏前期体验、剧情节奏与玩家心流

## 输入
- 本地游戏录屏路径
- 游戏名与版本
- 目标飞书电子表格 URL
- 唯一的新页签名称

## 固定拆解规则
- 分析层：`0–10min` 每 1 秒审阅，`10–20min` 每 5 秒审阅，`20min+` 每 10 秒审阅。
- 展示层：`0–30min` 每 1 分钟一个切片，`30–60min` 每 5 分钟一个切片，
  `1h+` 每 10 分钟一个切片，最后一片截到视频实际结束。
- 多段录屏按用户指定顺序拼成一条逻辑时间线；证据保留全局时间及源分段定位。
- 每片在中点抽取 1 张主截图；关键转折补 1–3 张证据图。
- 必须观看全部主截图；字幕、音频、对话、过场或上下文不足时回看原视频片段。

## 固定分析维度
1. 阶段目标：阶段名称与一句体验目标。
2. 任务链：该时间片的主要任务。
3. 核心循环：系统对玩家的教学及循环环节。
4. 渐进体验预期：期间解锁的系统、机制或界面。
5. 地图体验预期：期间解锁的地图或区域。
6. 经济体验预期：玩家等级、战力值与主城等级。
7. 剧情轴：剧情内容与叙事目的。

剧情轴必须标记高潮、低谷和心流高点，并关联时间与截图、字幕、音频或对话证据。
七维使用 `fact`/`inference` 对象；`fact` 为飞书与 HTML 精简展示，`inference` 仅供内部分析，不进入交付页面。
无有效信息允许空置，不使用“未观察到”或通用说明填充。剧情枚举为 `none/climax/low`，心流枚举为 `none/flow_peak`，但不在飞书显示值为 `none` 的说明。
证据只能引用所属片主图或证据图；阶段使用 `stage_id` 并保持名称、范围和相邻边界一致。

## 产出
- 统一事实源：校验后的 `analysis.json`。
- 多拆解查看器包：共享 `index.html`，每个游戏独占
  `data/<dataset-id>.json + screenshots/<dataset-id>/`；URL 使用
  `?dataset=<dataset-id>` 切换，禁止不同游戏共用或覆盖同一数据文件。
- 飞书：在用户指定电子表格中创建唯一名称的新页签，复刻参考时间轴与七维布局。

## 全局情绪与体验曲线
- `analysis.json` 根节点包含必填 `global_curves`，评分点与现有时间片一一对应。
- 情绪强度以剧情为主体：分别记录 `narrative_score`、`supporting_score` 与 `drivers`，最终分为
  `round(0.7 × narrative_score + 0.3 × max(narrative_score, supporting_score), 1)`。
- 其他刺激必须检查环境压力、暴风雪倒计时、紧迫目标、战斗、成长奖励和危机缓解；
  纯其他刺激最高贡献 1.5 分，负向高峰不代表剧情质量差。
- 体验强度按 `0～5` 人工评分，并记录玩法浓度、反馈密度、目标/挑战和打断情况。
- 体验趋势使用全部评分点和真实时间中点做线性回归；回归首尾差达到 `±0.5` 时判断上升或下降，否则持平。
- HTML 在同一 SVG 中叠加情绪强度、体验实际值和体验趋势；桌面图框为 480～600px，
  每个时间片使用覆盖绘图区全高的命中带，带内任意高度均可预览和点击。
- 曲线时间片选中后不常驻显示框线；桌面悬浮说明允许超出图框，移动端继续固定在视口底部。
- 七维信息继续保留在详情中，但不参与曲线加权；本轮不修改飞书页签结构。

## 全局 LOOP 流程
- `analysis.json` 根节点包含必填 `global_loops`，由 `scope`、`macro_loops`、`loop_families`、
  按时间排序的 `nodes` 和 `edges` 组成。
- 小 LOOP 的固定判断结构为“明确动机 → 连续行为 → 即时奖励 → 下一动机/再次投入”；每次形成完整闭环就独立拆分。
  单个教学步骤属于环内行为，一次性剧情或过渡使用线性节点。
- 每个 `micro_loop` 只能归属一个主要大 LOOP。寒霜大 LOOP 为聚落建设、战斗远征、英雄成长；
  其他系统影响通过 `cross_macro` 关系表达，不重复归属。
- 每个 `micro_loop` 必须显式填写唯一 `loop_family_id`；同类判断按玩法语义人工维护，
  不使用标题相似度自动推断。生产建筑归入建筑生产，法令系统与临时供暖强化分别独立成类。
- `primary` 按实际时间串联全部小 LOOP；同一大 LOOP 被插入内容打断后的再次出现使用 `macro_return`。
- HTML 在详情后、页面最后展示时间主链。每个小 LOOP 直接展开四个小节点；连续同类小 LOOP
  用同色分段背景带包住，被打断的分段用同色回流线连接，跨环关系从具体奖励/下一动机指向目标动机。
- 图例只淡化大 LOOP，不删除或重排节点。LOOP 悬浮窗完全移除，四段内容直接常驻卡片；
  `evidence_frames` 只保留于数据层用于校验和追溯。点击卡片联动首个可见时间片。
- 全局 LOOP 标题下、图例上方展示事件级总数、去重类型总数和每类出现/强化次数；
  `强化次数 = max(出现次数 - 1, 0)`，全部由节点数据派生。
- 时间片详情显示所有关联小 LOOP 标签，点击可滚动到对应卡片并短暂高亮；无线性节点标签，
  无关联时显示“当前时间片无关联 LOOP”。
- 寒霜固定拆成 17 个事件级小 LOOP（聚落 12、远征 3、英雄 2）和 6 个去重类型；
  六类出现次数为 8/2/3/2/1/1，强化次数为 7/1/2/1/0/0。P8 完整主城解锁为主体终点，
  P9 进入 SLG 大地图仅作图外虚线出口。
- 这是严格契约升级：旧 `chapters`、`entry`、`loop`、节点内 `accent` 和 `feedback` 数据必须迁移。

## 飞书边界与验收
- 仅支持新建页签；同名时只允许用户提供新名称或取消，不复用、不替换。
- 写入前执行只读 preflight 和 dry-run，正式写入时再次检查同名冲突。
- 完成标准包括内容回读、合并信息检查和导出 xlsx 的样式复核。
- `openpyxl` 缺失或标题样式、列宽、行高、合并、长文本可见性任一验证失败时，不得声称完成。
- `addSheet` 仅发送标题，超出默认尺寸时先调用独立的 `insert_dimension_range`；创建后任一步失败都尝试删除本次页签，并在错误 JSON 报告清理结果。
